# Introduction

Welcome to **zs-core-fhir-engine**.

This project is evolving toward a full healthcare interoperability platform for Bangladesh use cases, centered on **FHIR R5** and related standards.

## What exists today

- A broad Go FHIR model library (R4 + R5 resources).
- A runnable CLI with a lightweight FHIR API server.
- A basic terminology expansion endpoint.
- A published documentation site (GitHub Pages).

## What is being built next

To support real EMR/EHR/HIS production workloads, the project roadmap includes:

- Persistent data storage (database-backed server)
- Full FHIR conformance features
- Better terminology operations (ICD-11 and local code systems)
- HL7 v2 interoperability pipeline
- Security stack (OAuth2/OIDC, RBAC, audit)
